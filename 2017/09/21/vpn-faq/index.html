vpn-faq

发表于 2017-09-21 | 分类于 技术问题解答 | | 阅读次数

简述

在日常使用Vpn中会出现各种问题,下面我将列出一些常见问题的解决方法.

解决办法

VPN连接频繁掉线，无法连接服务器

运营商会随机干扰某些协议VPN端口，遇到这种情况时，请切换到其它协议使用即可。

错误 789/768 解决方法

1,点击 开始;
2,选择 控制面板
3,选择 管理工具，如果没有看到管理工具的选项，请在左上角，选择 经典视图;
4,选择服务，并查找 IPSec Service，在”属性”中设置”启动类型”为”自动”并启动服务.
（Win7/8下的服务名叫：IPSec Policy Agent）

错误 628/629/734 解决方法

出现这样的错误是加密模式选择错误。
请点击VPN→属性→安全→数据加密 在这个选项里，选择为可选加密，如果有看到MS-CHAPV2选项，请选中。
保存设置，再连接，就可以了

错误 868 解决方法

这种情况有2种原因：
由于本地网络的DNS服务器无法正确解析境外域名造成的。你需要将本地连接或无线连接的DNS服务器设置为208.67.222.222和8.8.8.8才能正确解析的（或者使用 114.114.114.114）。
服务器地址填写错误，VPN服务器地址（Internet地址）请在VPN提供商服务器列表里查看，建议使用域名连接。

错误 691 解决方法

帐号填写错误,请确认帐号/密码是否填写正确，密码区分大小写；
是否使用相应套餐可用服务器；
帐号是否到期；

错误 651 解决办法

1.服务器人数已满；
2.由于网络响应不及时，用户连续点击“连接”造成操作上的失误。
3.连接VPN时，当调制解调器(或其它连接设备)报告了一个错误，此时提示VPN错误称为VPN错误651。

VPN错误651处理方法:

1.如果是服务器人数已满,请换其他服务器进行连接；或者是你连接次数过多,导致无法正常跟服务器连接,请稍等两分钟再连接，或者重启下电脑。
2.如果是由于网络响应不及时，用户连续点击“连接”造成操作上的失误。关闭客户端软件，稍等片刻重新启动软件连接即可。
3.如果在网吧，请重启计算机，如果还解决不了联系网管。
4.虚拟网卡出现问题。解决办法重装虚拟机。
5.调制解调器出现故障。右击网上邻居属性，先暂时本地禁用一下，然后再启用本地，然后你把调制解调器的驱动卸载再重装一次，然后把你原来设置的拨号删除，重新再建一个，然后再次连接。

错误 721 解决办法

vpn错误721是电脑产生所导致的错误。 或者可以这样理解远程计算机没有响应。如果是从VPN商购买VPN，当登陆的购买VPN连接器时提示错误721时，直接联系VPN商解决。

通常地常见VPN错误721有以下两个；

1、 由于客户端电脑在局域网中并通过路由器上网所致。通熟点说是你本地路由问题。 主要原因是：VPN721错误是由于您的电脑处在您的电脑的上网方式是通过局域网中的路由器上网的造成的。例如：您的电脑是通过局域网中的路由器上网，在 局域网中就可能存在多台电脑，服务器就无法正确判断具体是哪台电脑（由于内网所有电脑的IP地址都是一样的），所以连接时会提示错误721。
2、该错误是由于客户端电脑跟服务器使用不同宽带服务商之间的直接互访问题所致。这个主要指的是网络宽带商的问题，一般的话是铁通宽带、长城宽带才会发生类似的问题；反之，是网通宽带、电信宽带很少出现这样的情况；

常见VPN错误721解决方法

1.单击“开始”，然后单击“运行”。
2.在“打开”框中，键入 regedit，然后单击“确定”。
3.在注册表编辑器中，找到以下子项

其中 <000x> 是 WAN 微型端口 (PPTP) 驱动程序的网络适配器。
4.在“编辑”菜单上，指向“新建”，然后单击“DWORD 值”。
5.键入 ValidateAddress，然后按 Enter。该值的默认设置为“1”（打开）；因此，您可以通过将其设置为“0”将其关闭。
6.退出注册表编辑器。
7.重新启动计算机。

错误 720 解决办法

vpn错误720定义是无法建立与远程计算机的连接。您可能需要更改此连接的网络设置。也可以简单理解成VPN服务器问题。
vpn错误720处理方法：
1.可能服务器问题，非用户问题，请向管理员反映；可能是服务器人过多，你可以尝试多连几次。
2.看看你电脑或者VPN服务器宽带连接是否正常。
3.还有另一种解决方法，重装TCP/IP协议:
（1）、开始——运行——regedit.exe，打开注册表编辑器，删除以下两个键：

（2）、用记事本打开%winroot%\inf\nettcpip.inf文件，找到：
[MS_TCPIP.PrimaryInstall]
Characteristics = 0xa0 <――――――把此处的0Xa0改为0x80 保存退出
（3）、打开本地连接的TCP/IP属性――-添加协议——从磁盘——浏览找到刚刚保存的 nettcpip.inf(%winroot%\inf\nettcpip.inf)文件，然后选择“TCP/IP协议”（不是选择那个TCP/IP 版本6）。
经过这一步之后，又返回网络连接的窗口，但这个时候，那个“卸载”按钮已经是可用的了。点这个 “卸载”按钮来把TCP/IP协议删除，然后重启一次机器。
（4）、重启后再照着第3步，重新安装一次TCP/IP协议便可。
（5）、再重启一次，这时应该可以了，可以照着需要，设置一下IP地址。

错误 769 解决办法

1.确保网络电缆已连接。即保证自己电脑可以顺利上网。
2.网络连接是否禁用。首先打开“我的电脑”→“控制面版”→“网络连接”，查看本地连接的是否处在 “禁用”状态，是的话只需双击本地连接，看到状态变为“已启用”即可。
3.网卡问题。若是连本地连接都没有的话，绝大部分情况是你网卡有问题；此时请检查网卡装好了没或网卡坏了。如果是网卡没装好重装即可。 如果是网卡坏了即时更换。例如：电脑中毒，把病毒删除之后出现VPN错误769代码，此时卸载网卡，重新搜索新硬件。重装网卡即可。
4.当本地能上网而连接不上VPN服务器。原因是当本机能上网并不代表你一定能连接到指定的VPN服务器，此时可以ping对应VPN服务器IP，如果 ping得通证明服务器没问题即可以连接上。导致的原因有VPN服务器没有开启或者你的电脑防火墙阻挡住VPN连接导致无法连接到目标。
5.硬件问题。例如：路由器不支持VPN，不过现在比较少，或者路由器自带防火墙阻挡了。
注意：出现VPN错误769代码，绝大多数是网卡被禁用，此时启用即可！造成的原因一般为：用户的误操作或一些防火墙软件、计算机病毒引起的。如果没有找到本地连接，一般是网卡的驱动程序丢了或计算机没有检测到网卡，此时需要重新安装即可。

错误 650 解决办法

在建立VPN时，远程计算机没有响应，断开连接，此时通常出现VPN错误650。
出现VPN错误650的原因：ADSL ISP服务器故障，网卡故障，非正常关机造成网络协议导致出错。
VPN错误650解决办法：检查ADSL信号灯是否能正确同步，检查网卡，删除所有网络组件重新安装网络。

错误 711 解决办法

v PN 错误711，导致原因相关服务没有开启,一般可以查阅以下3个服务Telephony、 Remote Access Connection Manager 、Remote Access Auto Connection Manager.这是Win7电脑系统里面算是一个比较常见的问题:
VPN错误711导致的原因：
1.检查网卡是否装好,重新安装网卡驱动；同时重新设置网络拨号连接。
2.检查下列服务是否被禁用了,如果被禁，将其改为手动。Telephony(电话服务)Remote Access Connection Manager(远程访问连接管理器)Remote Access Auto Connection Manager(远程访问自动连接管理器)
3.用管理员账号登陆，再打开C：\windows\system32\logfiles ，找到 wmi文件夹，在wmi文件夹上点右键，点选 获取管理员权限 ,重启即可修复。
4.开始-运行-输入 CMD -确定-在命令符后输入netsh winsock reset -回车，重启。
5.更改计算机名称。右击计算机－属性－高级系统设置－计算机名－更改－把更改里的计算机名改一个简短的，比如 china -确定。
6.换个系统版本重装系统。

vpn错误711解决方法：

1.启动Remote Access Connection Manager与Network Connections与Telephony服务。
2.启动方法一：开始->运行->在里面输入：services.msc ->（双击）Remote Access Connection Manager与Network Connections与Telephony ->启动这三项服务
3.启动方法二：新建一个文本文件（后缀为.txt）的文件，将下面的内容复制进去：

点击菜单->文件->另存为；文件类型选“所有文件”；文件名“服务.bat”；保存好后双击“服务.bat”，即可完成系统服务设置。

错误 634 解决办法

远程访问VPN服务器不能在网络上注册你的计算机名称，这一般随着NetBIOS协议一起出现，但也可能随着tcp/ip或者ipx一起出现。通常，当地址已经在网络上使用时，会发生vpn错误634。如果对这方面不是很专业的请与系统管理员联系处理vpn错误634。

错误 678 解决办法

1.可能是vpn服务器人数过多，请尝试多连接几次 ；
2.可能是你装反了客户端，请重新安装客户端；
3.从上面定义不难看出，调制解调器或其他设备故障，检查号码，再拨号。如果是虚拟专用网络 (VPN) 连接，则检查目标服务器的主机名或 IP 地址，然后再次尝试连接。还要确保电话线(网线)插在调制解调器（网卡）的正确插槽中。
4.VPN pptp远程拨号错误678；
5.如果您是购买VPN用户直接联系商家解决；

错误 619 解决办法

vpn错误619是指端口已断开连接。也可以理解成已经断开了VPN连接。

vpn错误619原因以及解决办法:

1.vpn错误619；这种情况大数多原因为多人连接VPN服务器异常中断，因绝大部分VPN服务器限制一个帐户同时只有一个人使用，所以一般来说一个VPN账号只能一个人使用，多人登陆就会出现这种情况。
解决办法：VPN只能一个人登陆VPN连接器。
以下vpn错误619两点基本是商家需要解决的，如果以上第一点排除之外普通VPN用户直接联系商家解决即可；
2.这种情况大数多原因为客户机连接Internet的网关（如家庭宽带路由或公司上网网关路由或防火墙）NAT-T功能关闭或对VPN支持性不好，主要是对GRE及PPTP协议的NAT-T不支持。
解决办法：打开网关路由的NAT-T功能，如果还是出现错误，则需要更换网关设备，现在大多数设备已经支持。
3.（这点是VPN服务商的问题）有些VPN服务器没有设置异常断线检查功能也可能导致用户一旦异常断开后很长时间不能连接。
解决办法：在服务器上设置异常断线检查程序或功能。

错误 800 解决办法

vpn错误800是指不能建立VPN连接，VPN服务器可能不能到达，或者此连接的安全参数没有正确配置。

VPN800错误解决方法：

一般来说你如果是购买VPN用户的话，无需下面的步骤，直接联系VPN提供商解决即可。
1.IP错误。检查配置中的“目的地的主机名或IP地址”使用的是域名而不是IP地址，由于VPN隧道需要定期进行维护，我们有可能变更VPN隧道服务器的IP地址，但域名不会改变。
2.您所在网络与我们的VPN隧道服务器没有正确的通道，请尝试更换配置中的“目的地的主机名或IP地址”，将其调整为我们的登陆服务器IP地址或者域名。
3.临时性故障，（如果您是用户联系VPN商家解决即可）多半是由于DNS服务器繁忙无法对服务器IP地址，或者域名的名字进行解析所引起，可以使用以下操 作：开始->运行->打开：cmd->确定，执行命令ipconfig /flushdns后再进行VPN连接尝试。
4.重启 电脑，删除原来建好的，重新一个新的VPN连接。 由于配置异常造成的无法连接，虽然用户没有做过任何的配置修改，但由于系统内部的故障会导致配置（注册表信息）出现异常，这是Windows系统中常见的 问题。 处理方法是删除原VPN隧道连接的配置，重新建立一个新的VPN隧道连接即可。
5.检查连接的安全参数配置与配置要求一致。
6.防火墙太严。您机器上的防火墙规则设置过于严格，导致无法对外进行连接，请调整或关闭所有防火墙再进行尝试。
7.如果有安装家庭网关的用户，也建议重启一下家庭网关设备。

错误 756 解决办法

在连接VPN代理时，VPN连接正在运行，但无法正常连接称为VPN错误756一般咱们发现提示的错误为：VPN错误756,已经拨了这个连接。但有时一开始会伴随出现VPN错误800或者VPN错误619的发生。

VPN错误756原因可能由以下几点造成的：

1.在连这个VPN之前，可能已连接了另一个连接，但没有正常有效断开关闭。
2.这个VPN的帐号未正关闭，已经或仍在使用中。就像家里ADSL帐号没正常关闭,挂着，马上再连就连不上了,隔一会连接就OK了。
解决方法如下:
1.重启下电脑；如果电脑不能有效关闭先前的VPN连接，此时重启下电脑。
2.重启路由器。
3.也有可能是这种情况多数为VISTA或者windows 7系统用户，解决方法为：请用鼠标右键点软件，选择“以管理员身份运行”即可。

  1. 当系统为XP，出现VPN错误756是因为你在登陆过程中非正常中断，导致账号连接出现错误，出现这种情况最好的办法就是：点击网上邻居—–反键—–属性 —–虚拟网络 ，在虚拟网络里面看到 VPN网络连接断开 ，删除后重启计算机，然后再连接；如果无法删除，直接重启计算机，然后再删除后连接。

结语

在使用VPN时很多问题会导致其无法正常使用,以上方法仅供参考.
